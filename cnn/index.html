How convolution works — the math

\[ Z_{i,j} = \sum_{m}\sum_{n} X_{i+m,\,j+n} \cdot K_{m,n} \] \[ \text{Output size} = (H - k + 1)\times(W - k + 1) \] \[ = (5-3+1)\times(5-3+1) = \mathbf{3\times 3} \]

\(H, W\): input height/width; \(k\): kernel size (here \(k=3\)); \(m, n\): kernel indices.

Step 3 — ReLU Activation Non-linearity

Apply \(\text{ReLU}(x) = \max(0,\, x)\) element-wise. Negative values become 0.

After ReLU \((\mathbf{A})\)

Why ReLU?

\[ A_{i,j} = \text{ReLU}(Z_{i,j}) = \max(0,\, Z_{i,j}) \]

Without non-linearity, stacking linear layers is still linear. ReLU introduces non-linearity so the network can learn complex patterns.

Step 4 — Max Pooling Spatial Reduction

Slide a \(2{\times}2\) window (stride 1) and take the maximum value in each region.

After ReLU \((\mathbf{A})\) — \(3{\times}3\)

→ MaxPool →

Pooled Output \(\mathbf{P}\)

\(3{\times}3\) input, \(2{\times}2\) pool, stride 1 → \(2{\times}2\) output.

What pooling does — the math

\[ P_{i,j} = \max_{0 \le m,n < k}\, A_{i{\cdot}s+m,\; j{\cdot}s+n} \] \[ \text{Output size} = \left\lfloor \frac{H - k}{s} \right\rfloor + 1 \]

\(k\): pooling window size (here \(k=2\)); \(s\): stride (here \(s=1\)); \(m, n\): window indices.

Pooling reduces spatial dimensions (fewer parameters downstream) and provides translation invariance — small shifts in the input give the same output.

Step 5 — Flatten \(2\text{D} \to 1\text{D}\)

Reshape the 2D feature map \(\mathbf{P}\) into a 1D column vector \(\mathbf{v}\) — the input to the fully-connected layer.

Pooled \(\mathbf{P}\) (2D)

→ Flatten →

Vector \(\mathbf{v}\) (1D)

Row-major flattening

\[ \mathbf{v} = \text{flatten}(\mathbf{P}) \] \[ v_{i \cdot W + j} = P_{i,j} \] \[ \mathbf{v} \in \mathbb{R}^{N}, \quad N = H \cdot W \]

\(W\): number of columns in \( \mathbf{P} \); \(H\): number of rows; \(N\): total elements.

This is a reshape operation — no weights, no computation. It simply unrolls the matrix row by row into a vector so a dense layer can process it.

Matrix multiplication — the math

\[ \mathbf{z} = \mathbf{W}\mathbf{v} + \mathbf{b} \] \[ z_k = \sum_{j} W_{k,j}\, v_j + b_k \] \[ \hat{y}_k = \text{softmax}(\mathbf{z})_k = \frac{e^{z_k}}{\displaystyle\sum_{j} e^{z_j}} \]

\(\mathbf{W}\): weight matrix; \(\mathbf{v}\): flattened input; \(\mathbf{b}\): bias; \(k\): output class index.
